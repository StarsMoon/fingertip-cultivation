# 《指尖修仙》游戏设计文档 (GDD)

> 一款面向上班族碎片时间的微信休闲小游戏  
> 一指操作 · 两分钟一局 · 修仙割草 · 越打越爽

---

## 1. 游戏概述

| 字段 | 内容 |
|---|---|
| **游戏名称** | 指尖修仙 |
| **类型** | 休闲 Roguelike 幸存者 (Casual Survivor) |
| **平台** | 微信小游戏 (Canvas 2D) |
| **单局时长** | 60~120 秒 |
| **操作方式** | 单指拖拽移动 |
| **网络要求** | 单机可玩，排行榜需联网 |
| **包体目标** | < 4MB |

### 一句话描述

> 你是一位修炼中的仙人，妖魔从四面八方涌来——只需一指滑动走位，灵力自动攻击，击杀妖魔突破境界，解锁更强的功法，一局两分钟，越打越爽。

### 为什么做这个

| 竞品 | 问题 | 我们的解法 |
|---|---|---|
| 弹壳特攻 / 幻兽帕鲁幸存者 | 操作复杂、局时长、包体大 | 一指操作、2分钟局、<4MB |
| 合成大西瓜 / 羊了个羊 | 无成长线，长留差 | Roguelike Build + 永久养成双线 |
| 跳一跳 / 2048 | 爽感不足，无视觉冲击 | 全屏弹幕割草，视觉极度满足 |
| 现有修仙小游戏 | 纯挂机，无操作感 | 有操作+有养成，操作收益明显 |

## 2. 目标用户画像

### 核心用户：摸鱼上班族

| 特征 | 设计对应 |
|---|---|
| 每次摸鱼 1~3 分钟 | 单局 60~120s，随时可停 |
| 只能单手操作 | 单指拖拽，无需多指 |
| 随时可能被打断 | 无惩罚退出，进度即时保存 |
| 需要即时爽感释放压力 | 全屏割草 + 数字飞涨 + 音效反馈 |
| 和同事有比较心理 | 部门排行榜、每日挑战 |
| 反复玩同一款游戏 | Roguelike 随机性 + 永久养成 |

### 次要用户：学生、通勤族

- 地铁/课间碎片时间
- 社交分享需求更强
- 更愿意看广告换资源

## 3. 核心玩法循环

```
┌─────────────────────────────────────────────────────┐
│                    单局循环 (1~2 min)                  │
│                                                       │
│  进入关卡 → 拖拽走位 → 灵力自动攻击 → 击杀妖魔         │
│      ↑                                    ↓          │
│  拾取经验 ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←          │
│      ↓                                               │
│  升级选功法 (三选一) → 新能力 → 继续战斗                │
│      ↓                                               │
│  境界突破 (全屏特效) → 更强妖魔 → 生存到时间结束         │
│                                                       │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                  元游戏循环 (跨局)                      │
│                                                       │
│  结算 → 获得灵石/材料 → 永久升级(攻防/新角色/新功法)    │
│      → 解锁新关卡 → 下一局更强                         │
│                                                       │
└─────────────────────────────────────────────────────┘
```

### 操作方式

- **唯一操作**：单指按住屏幕拖拽 → 角色跟随移动
- **无按钮、无技能键**：所有攻击自动触发
- **暂停**：双击屏幕或抬起手指 1.5s 后自动暂停

> 设计哲学：操作极简到无法更简，深度来自 Build 选择而非操作复杂度

## 4. 战斗系统

### 4.1 角色属性

| 属性 | 说明 | 成长方式 |
|---|---|---|
| 攻击力 | 每次伤害基础值 | 永久升级 + 功法加成 |
| 攻速 | 每秒攻击次数 | 功法加成 |
| 灵力范围 | 自动攻击半径 | 功法加成（核心成长维度）|
| 移速 | 角色移动速度 | 永久升级 |
| 最大生命 | 承受伤害上限 | 永久升级 |
| 拾取范围 | 经验珠吸收半径 | 功法加成 |

### 4.2 自动攻击系统

角色持续对灵力范围内最近/随机敌人发动攻击，玩家只需控制走位让更多敌人进入范围。

**初始攻击**：掌心雷 — 向最近敌人发射一道直线雷光

### 4.3 升级 & 功法选择（核心乐趣）

每次升级弹出 3 个功法选项（三选一），功法叠加产生质变。

#### 功法设计原则

1. **每个功法单独可感知** — 拿到立刻看到变化
2. **叠加有化学反应** — 1+1>2 的组合感
3. **有取舍** — 不可能一局全拿

#### 功法分类

| 类别 | 示例 | 效果 |
|---|---|---|
| **新弹幕** | 御剑术 | 身周 4 把飞剑环绕旋转 |
| | 火球术 | 向随机方向发射火球，爆炸 AOE |
| | 冰锥阵 | 地面生成冰锥，踩中敌人减速+受伤 |
| | 雷链 | 闪电在敌人间跳跃，最多连 5 个 |
| **强化** | 暴击诀 | 攻击 20% 概率双倍伤害 |
| | 穿透符 | 弹幕穿透 +1 个敌人 |
| | 弹射印 | 弹幕击中后反弹到下一个敌人 |
| **辅助** | 灵吸大法 | 拾取范围 +50% |
| | 铁骨功 | 受到伤害 -30% |
| | 疾风步 | 移速 +40% |

#### 功法进化（同名功法重复出现时升级）

| 阶段 | 御剑术 | 火球术 |
|---|---|---|
| Lv1 | 2 把剑旋转 | 发射 1 颗火球 |
| Lv2 | 4 把剑，伤害 +50% | 发射 2 颗火球 |
| Lv3 | 6 把剑，范围 +30% | 火球爆炸范围 +100% |
| Lv4 | 8 把剑，附加雷属性 | 火球留下燃烧地面 |

#### Build 组合示例

| Build 名 | 功法组合 | 效果 |
|---|---|---|
| 满屏飞剑 | 御剑术 Lv3 + 穿透符 + 弹射印 | 飞剑穿透+弹射，全屏刮 |
| 火神降临 | 火球术 Lv3 + 暴击诀 + 范围强化 | 巨大火球全屏爆炸 |
| 闪电链王 | 雷链 Lv3 + 弹射印 + 暴击诀 | 雷电全屏跳，暴击跳更远 |
| 无限冰封 | 冰锥阵 Lv3 + 减速强化 + 范围强化 | 地面全冰，敌人寸步难行 |

### 4.4 境界突破（局内里程碑）

击杀足够妖魔触发「境界突破」，全屏金光特效，短暂无敌，全属性提升一档。

| 境界 | 所需经验 | 效果 |
|---|---|---|
| 练气 → 筑基 | Lv5 | 攻击力 +30%，灵力范围 +20% |
| 筑基 → 金丹 | Lv10 | 攻速 +25%，移速 +15% |
| 金丹 → 元婴 | Lv15 | 全属性 +20%，新弹幕槽位 +1 |
| 元婴 → 化神 | Lv20 | 全属性 +40%，弹幕自动追踪 |

> 每次突破伴随震撼全屏特效——这是玩家最爽的时刻

### 4.5 妖魔设计

| 类型 | 行为 | 威胁 |
|---|---|---|
| 小妖 | 直线走向玩家，移动慢 | 炮灰，提供经验 |
| 飞妖 | 从屏幕外斜线飞入 | 突然出现，需走位 |
| 射手妖 | 站在远处发射弹幕 | 需要优先接近击杀 |
| 坦克妖 | 移动慢但血厚 | 挡路，需集火 |
| 自爆妖 | 接近后爆炸 | 需提前击杀或走位躲避 |
| **Boss** | 每 60s 出现，多阶段 | 需要合理 Build 应对 |

## 5. 关卡系统

### 5.1 关卡结构

| 关卡 | 主题 | 时长 | Boss |
|---|---|---|---|
| 青云山 | 入门，妖魔弱 | 60s | 狐妖 |
| 落霞谷 | 引入射手妖、飞妖 | 90s | 蛇妖 |
| 幽冥涧 | 引入自爆妖 | 90s | 鬼将 |
| 天魔殿 | 妖魔密集，弹幕多 | 120s | 魔尊 |
| 混沌虚空 | 无尽模式，越来越难 | ∞ | 每 60s 一个 Boss |

### 5.2 难度曲线

```
难度 ↑
     │                    ╱ Boss
     │               ╱───╱
     │          ╱───╱
     │     ╱───╱
     │╱───╱
     └─────────────────────→ 时间
     0    30    60    90   120s
```

- 前 20s：稀疏小妖，让玩家熟悉本局 Build
- 20~60s：妖魔种类增加，密度上升
- 60s+：Boss 出现 + 精英妖混合
- 90s+：极限生存，屏幕满是妖魔和弹幕

## 6. 养成系统（跨局成长）

### 6.1 永久升级（灵石消耗）

| 升级项 | 每级效果 | 上限 | 灵石消耗 |
|---|---|---|---|
| 攻击力 | +5% | Lv20 | 递增 50/100/200... |
| 最大生命 | +8% | Lv20 | 递增 |
| 移速 | +3% | Lv10 | 递增 |
| 初始灵力范围 | +5% | Lv10 | 递增 |
| 经验获取 | +5% | Lv10 | 递增 |
| 幸运值（稀有功法概率）| +3% | Lv10 | 递增 |

### 6.2 角色解锁

| 角色 | 初始弹幕 | 特殊被动 | 解锁条件 |
|---|---|---|---|
| 青云弟子 | 掌心雷 | 无 | 初始 |
| 剑修 | 御剑术 | 暴击率 +10% | 灵石 ×500 |
| 火修 | 火球术 | 攻击力 +15% | 灵石 ×800 |
| 冰修 | 冰锥阵 | 减速光环 | 通关落霞谷 |
| 雷修 | 雷链 | 连锁 +1 | 通关幽冥涧 |
| 魔修 | 暗影弹 | 吸血 5% | 通关天魔殿 |

### 6.3 功法图鉴

- 收集过的高级功法记录在图鉴中
- 图鉴收集度 → 全局属性加成
- 刺激玩家尝试不同 Build

## 7. 社交系统

### 7.1 排行榜

| 排行榜 | 说明 |
|---|---|
| 好友排行 | 微信好友最高分 |

- 每天固定种子生成的关卡（所有人相同）

- 结算页一键分享到微信好友/群
- 分享图包含：境界、Build、分数、击败妖魔数
- 分享解锁：每日首次分享获 100 灵石

### 9.1 美术方向

- **风格**：国风水墨 + 赛博霓虹混搭
- **角色**：Q 版修仙人物，2 头身比例

- **特效**：境界突破全屏金光，功法升级华丽粒子

### 9.2 色彩方案

## 10. 音频设计

### 10.1 音效

| 事件 | 音效风格 |
|---|---|
| 自动攻击 | 清脆「铮」声，不同功法音色不同 |
| 击杀妖魔 | 短促「噗」，连杀音调递升 |
| 升级 | 仙乐上行琶音 |
| 境界突破 | 钟鸣 + 管弦齐奏 |
| 受伤 | 沉闷「咚」 |
| 死亡 | 渐弱和弦 |
| 拾取经验 | 微弱「叮」(大量时叠加成和弦) |

### 10.2 背景 BGM

- 古风电子混搭（古筝 + 电子节拍）
- 随境界提升 BGM 层级增加（练气→单乐器，化神→全编制）
- 60 BPM 慢节奏，不干扰摸鱼

### 10.3 静音友好

- 所有核心信息有视觉反馈，不依赖音效
- 公共场合静音游玩完全可行

## 11. 技术方案

### 11.1 技术栈

| 组件 | 选型 | 原因 |
|---|---|---|
| 运行时 | 微信小游戏 Canvas 2D API | 官方支持，兼容性最好 |
| 语言 | TypeScript → 编译为 JS | 类型安全，开发效率 |
| 物理 | 无物理引擎，纯数学碰撞 | 包体小，性能好 |
| 动画 | 手写补间 + 粒子系统 | 轻量可控 |
| 状态管理 | 简单观察者模式 | 游戏规模不需要 Redux |
| 构建 | 自定义 webpack/rollup | 按需打包，控制体积 |
| 调试 | 微信开发者工具 + Chrome DevTools | 标准 WeChat 开发流程 |

### 11.3 性能目标

| 指标 | 目标 |
|---|---|
| 帧率 | 稳定 60fps，同屏 200 妖魔不卡 |
| 内存 | < 150MB |
| 包体 | < 4MB（首包，资源按需下载） |
| 冷启动 | < 2s 进入主菜单 |
| 热重开 | < 0.5s 开新一局 |

### 11.4 性能策略

- **对象池**：弹幕、妖魔、粒子全部池化，零 GC 压力
- **空间哈希**：碰撞检测用网格分区，O(n) 而非 O(n²)
- **离屏渲染**：静态 UI 层缓存为位图，不每帧重绘
- **分级渲染**：远处妖魔降低帧率、简化动画
- **对象上限**：同屏弹幕上限 500，超出的不渲染只算伤害

## 15. 数值设计（核心公式）

### 伤害公式

```
最终伤害 = 基础攻击力 × (1 + 攻击力加成%) × 功法倍率 × 暴击倍率
暴击倍率 = 暴击判定 ? 2.0 : 1.0
```

### 经验公式

```
升级所需经验 = 10 × Level^1.5
妖魔经验掉落 = 妖魔基础经验 × (1 + 经验加成%)
```

### 妖魔生成公式

```
生成间隔 = max(0.3, 基础间隔 - 游戏时间 × 加速率)
同屏上限 = 50 + 游戏时间 × 2 (封顶 200)
妖魔血量 = 基础血量 × (1 + 关卡等级 × 0.5)
```

### 灵石收益

```
灵石 = 击杀数 × 1 + 境界数 × 10 + 通关奖励 × 是否通关
```

## 16. 风险 & 应对

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| 全屏弹幕卡顿 | 高 | 体验毁灭 | 对象池 + 粒子上限 + 分级渲染 |
| 功法平衡性差 | 高 | Build 单调 | 数据埋点 + 快速迭代 + 社区反馈 |
| 同类竞品先发 | 中 | 用户分流 | 修仙题材差异化 + 社交裂变 |
| 审核不通过 | 低 | 上线延迟 | 提前了解小游戏审核规范 |
| 留存率低 | 中 | 商业化失败 | 每日挑战 + 排行榜 + 养成线 |
